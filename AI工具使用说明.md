## 4. 遇到的问题及如何通过 AI 解决

### 问题 1：描述文字太小
- **现象：** 初版生成后，「关于我」正文和项目描述字体只有 14px，阅读体验差
- **解决：** 直接告诉 AI「字体好像有点小」，AI 快速定位到 6 处 CSS 规则并将字号提高 1-2 档

### 问题 2：项目卡片缺少交互深度
- **现象：** 初版只有静态卡片，点击无反应
- **解决：** 提出「点击项目展示详细信息」，AI 设计了完整的 Modal 弹窗系统，包括架构流程图、核心亮点列表、图片展示、链接占位

### 问题 3：板块顺序不合理
- **现象：**「关于我」放在第二屏，用户觉得应该先看技能和项目
- **解决：** 告诉 AI 调整顺序，AI 同步更新了 HTML 结构和导航栏链接

### 问题 4：国内网络无法直连 GitHub
- **现象：** HTTPS 推送代码时报 `Failed to connect to github.com port 443`
- **解决：** AI 排查网络后改用 SSH 方案——生成了 Ed25519 密钥对，指导将公钥添加到 GitHub 设置，通过 SSH 协议成功推送

### 问题 5：项目截图需要集成到详情弹窗
- **现象：** 弹窗中只有「点击上传或拖入图片」占位符
- **解决：** 将截图放入 src 文件夹后，AI 自动识别文件名，更新 JS 数据中的 images 数组，并将渲染逻辑从占位符改为真实的 `<img>` 标签 + 灯箱放大

## 总结

本次测试完整展示了 **"需求理解 → AI 驱动开发 → 人工审核反馈 → 迭代优化 → 上线部署"** 的完整工作流。核心体会到：

- **AI 是杠杆，不是替代品**：AI 能在几分钟内生成完整的 HTML/CSS/JS 代码，但架构决策（板块设计、交互逻辑）、审美判断（配色、间距）、和质量把关（内容准确性）仍然需要人来做出
- **迭代式对话**：一次说清楚所有需求不容易，更好的方式是先做出初版，然后基于实际效果不断提出修改意见
- **全流程交付**：从代码生成到外网部署，AI 可以覆盖每一个环节，大大缩短交付周期
